## 1. 🧠 Пам'ять у рамках однієї сесії (In-Memory Chat History)

### 📌 Мета
Зараз у `handle_commands()` ([src/ollama.py](src/ollama.py#L53-L80)) кожен запит відправляється окремо через `context.build_prompt(user)`. Модель не пам'ятає попередніх реплік. Необхідно додати підтримку діалогу в межах поточного запуску.

### 🎯 Вимоги
- **In-Memory:** Історія зберігається лише в RAM під час роботи `owl` та очищається при виході (`exit`/`quit`/`Ctrl+C`).
- **Формат Ollama API:** Передача масиву `messages` з ролями `system`, `user`, `assistant`.
- **Сумісність:** Системний контекст з [Context](src/context.py#L14-L84) передається як перше `system`-повідомлення або оновлюється без втрати історії.

### 🛠 План реалізації
1. Змінити `generate()` або додати `generate_chat(messages: list[dict]) -> str` у [src/ollama.py](src/ollama.py#L29-L50).
2. У `handle_commands()` ([src/ollama.py](src/ollama.py#L53-L80)) ініціалізувати список `messages`, додавати туди репліки користувача та відповіді моделі.
3. *Опціонально:* Обмежити довжину історії (наприклад, останні 10–20 повідомлень) для економії токенів.

### 📋 Чекліст
- [ ] Оновити `generate` у [src/ollama.py](src/ollama.py) для роботи з масивом `messages`.
- [ ] Реалізувати збереження історії в циклі `handle_commands()`.
- [ ] Адаптувати передачу контексту з [src/context.py](src/context.py) у `system`-роль.
- [ ] Протестувати пам'ять діалогу та переконатися в очищенні при перезапуску.

## 2. 🦉 🎓 English Buddy (Інтерактивний викладач англійської)

### 📌 Мета
Режим персонального репетитора та співрозмовника англійською мовою з **довгостроковим (persistent) збереженням** проблемних слів та граматичних помилок між запусками програми.

### 🎯 Вимоги
- **Режим практики:** Команда `/english` (або `/buddy`) у [src/ollama.py](src/ollama.py#L60) чи автовиявлення. Модель спілкується англійською, виправляє помилки та пояснює правила.
- **Персистентне сховище:** Збереження незнайомих слів, фраз, помилок та дат для інтервального повторення у файл (наприклад, `~/.local/share/owl-droid/english_vocabulary.json`).
- **Модуль контексту:** Створити `EnglishBuddyContextModule` для [src/context.py](src/context.py#L7-L11), який підвантажує слова на повторення в системний промпт при старті.
- **Інтеграція:** Пропозиція «слова дня» в привітанні ([build_greeting_prompt()](src/context.py#L50-L58)) та використання вивчених слів у діалогах.

### 🛠 План реалізації
1. Створити клас `VocabularyStorage` (`src/modules/english_buddy/storage.py`) для CRUD-операцій з JSON-сховищем.
2. Реалізувати `EnglishBuddyContextModule` (`src/modules/english_buddy/buddy_context.py`) та додати в `MODULES` у [src/context.py](src/context.py#L7-L11).
3. Додати CLI-команди (`word add <word> <trans>`, `buddy list`) або автоматичний парсинг тегів від моделі (наприклад, `[LEARN: word="...", topic="..."]`).
4. Додати команду `/quiz` для вправ і міні-тестів.

## 3. 🦉 📚 Локальна Вікі та RAG (База знань: кава, чай тощо)

### 📌 Мета
Підключення локальної бази знань (Wiki) для надання точних експертних відповідей на обрану тематику (наприклад, **альтернативні методи заварювання кави чи чайні церемонії** — рецепти V60, Аеропрес, Матча, пропорції, температура), уникаючи галюцинацій завдяки RAG.

### 🎯 Вимоги
- **Структура Вікі:** Зберігання структурованих Markdown/JSON файлів у виділеній теці (наприклад, `~/.local/share/owl-droid/wiki/coffee/` чи `docs/wiki/`).
- **Пошук (Retriever):** Базовий пошук за ключовими словами/назвами файлів або семантичний пошук через локальні векторні ембединги Ollama.
- **Модуль Вікі:** Створення `WikiContextModule` або обробка команд `/wiki <запит>`, `/recipe <назва>`. При виявленні релевантного запиту інструкції з вікі динамічно додаються до системного промпту.
- **CLI-команди:** Перегляд тем (`wiki list`, `recipe list`) та читання/додавання статей (`wiki show <title>`, `wiki add <title>`).

### 🛠 План реалізації
1. Додати шлях `WIKI_PATH` у [src/config.py](src/config.py) та `.env` (за замовчуванням `~/.local/share/owl-droid/wiki/`).
2. Створити пошуковий клас `WikiRetriever` (`src/modules/wiki/retriever.py`) для пошуку файлів за ключовими словами чи ембедингами.
3. У `handle_commands()` ([src/ollama.py](src/ollama.py#L58-L78)) інтегрувати RAG: при відповідному запиті знаходити статті у вікі та передавати їх у промпт.

### 📋 Чекліст
- [ ] Додати параметр `WIKI_PATH` у [src/config.py](src/config.py) та створити набір тестових рецептів (кава/чай).
- [ ] Реалізувати клас `WikiRetriever` (`src/modules/wiki/retriever.py`).
- [ ] Додати команди `/wiki` та `/recipe` у [src/ollama.py](src/ollama.py#L58-L78).
- [ ] Інтегрувати підвантаження статей у промпт (RAG).
- [ ] Протестувати точність відповідей за локальними інструкціями.
